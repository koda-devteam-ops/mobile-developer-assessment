# Requirements

## Scenario

You are building a mobile application for a Client Project Tracker used by a digital agency.

The app should allow users to manage client projects on mobile devices.

You may connect to a backend API OR use local mock data.

---

## Project Model

Each project contains:

- ID
- Client Name
- Project Name
- Description
- Status
- Priority
- Start Date
- Due Date

---

## Required Screens

### 1. Project List Screen

Display all projects.

Show:

- Client Name
- Project Name
- Status
- Priority
- Due Date

---

### 2. Create Project Screen

Allow users to create a new project.

---

### 3. Edit Project Screen

Allow users to update project details.

---

### 4. Project Details Screen

View full project information.

---

## Required Features

- CRUD operations (via API or local storage)
- Form validation
- Loading states
- Error handling

---

## Technical Requirements

- You may use any mobile framework:
  - React Native
  - Flutter
  - Native Android / iOS
- Architecture approach is up to you

---

## Bonus (Optional)

- Offline support
- Search
- Filtering
- Dark mode
- State persistence
- Push notification simulation
- Clean navigation structure
